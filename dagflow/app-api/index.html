    App API and Operations

    App management procedures live in <app_name>.app_public. APP_ADMIN can call all procedures on this page. APP_EDITOR can call only update_code_location_image.

    Core Services

    start_app

    • Signature: 

      start_app()
      Creates the Dagflow compute pools and starts or updates the core.postgres and core.dagster services.

    • Arguments: none.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.start_app();

    grant_callback

    • Signature: 

      grant_callback(privileges ARRAY)
      Retries the same startup callback that Snowflake invokes when required app privileges are granted. Use this only when startup did not happen automatically after granting CREATE COMPUTE POOL and BIND SERVICE ENDPOINT.

    • Arguments:

    Argument Required Description
    privileges Yes Array of granted privilege names. The app starts only when the array contains both CREATE COMPUTE POOL and BIND SERVICE ENDPOINT.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.grant_callback([
  'CREATE COMPUTE POOL',
  'BIND SERVICE ENDPOINT'
]);

    service_status

    • Signature: 

      service_status()
      Shows services in the app core schema.

    • Arguments: none.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.service_status();

    dagster_containers

    • Signature: 

      dagster_containers()
      Shows containers for the core.dagster service.

    • Arguments: none.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.dagster_containers();

    dagster_logs

    • Signature: 

      dagster_logs(
  container_name STRING,
  line_count NUMBER
)
      Returns logs from a core.dagster container.

    • Arguments:

    Argument Required Description
    container_name Yes Dagster service container name. Use webserver or daemon.
    line_count No Number of log lines to return. Defaults to 100 when NULL.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.dagster_logs('webserver', 200);
CALL dagflow_app.app_public.dagster_logs('daemon', NULL);

    Code Locations

    add_code_location

    • Signature: 

      add_code_location(
  code_location_name STRING,
  blocking BOOLEAN
)
      Creates a new code-location service in the app CODE_LOCATIONS schema with Dagflow's bundled placeholder image. This is a create-only operation and fails if the service already exists. Pass FALSE to return after requesting service creation, or TRUE to wait for readiness. Use update_code_location_image to provide the real user code-server image.

    • Arguments:

    Argument Required Description
    code_location_name Yes Code location name. Must start with a letter and contain only letters, numbers, and underscores. Dagflow stores it uppercase.
    blocking Yes When TRUE, waits up to 5 minutes for the placeholder code-server container to report READY. When FALSE or NULL, returns after requesting service creation.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.add_code_location('finance', FALSE);

    Then provide the real user code-server image:

    CALL dagflow_app.app_public.update_code_location_image(
  'finance',
  '/DAGFLOW_CONSUMER_DB/CODE/CONSUMER_REPO/finance-dagster:2026-06-01',
  'finance.definitions',
  FALSE
);

    Use blocking when automation should fail if the placeholder code server does not become ready:

    CALL dagflow_app.app_public.add_code_location(
  'finance',
  TRUE
);

    update_code_location_image

    • Signature: 

      update_code_location_image(
  code_location_name STRING,
  image STRING,
  module_name STRING,
  blocking BOOLEAN
)
      Updates the image and module name for an existing code-location service. Pass FALSE to return after requesting service update, or TRUE to wait for readiness.

    • Arguments:

    Argument Required Description
    code_location_name Yes Existing code location name. Must start with a letter and contain only letters, numbers, and underscores.
    image Yes New Snowflake image repository path. Empty strings are not allowed.
    module_name Yes Python module containing Dagster definitions in the new image, for example finance.definitions.
    blocking Yes When TRUE, waits up to 5 minutes for all code-server containers to report READY. When FALSE or NULL, returns after requesting service update.

    • Required role: APP_ADMIN or APP_EDITOR.

    • Example: 

      CALL dagflow_app.app_public.update_code_location_image(
  'finance',
  '/DAGFLOW_CONSUMER_DB/CODE/CONSUMER_REPO/finance-dagster:2026-06-01',
  'finance.definitions',
  FALSE
);
      Use blocking when automation should fail if the updated code-server container does not become ready: 
      CALL dagflow_app.app_public.update_code_location_image(
  'finance',
  '/DAGFLOW_CONSUMER_DB/CODE/CONSUMER_REPO/finance-dagster:2026-06-01',
  'finance.definitions',
  TRUE
);

    remove_code_location

    • Signature: 

      remove_code_location(code_location_name STRING)
      Drops an existing code-location service. This removes it from Dagster after workspace refresh.

    • Arguments:

    Argument Required Description
    code_location_name Yes Code location name to drop. Must start with a letter and contain only letters, numbers, and underscores.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.remove_code_location('finance');
      Dagflow uses the registered code-server image as the run-worker image, so the image must include dagster-postgres and the dependencies needed to execute runs.

    Restart and Suspend

    restart_dagster

    • Signature: 

      restart_dagster()
      Suspends and resumes the core.dagster service. Use this when the control plane needs a manual restart.

    • Arguments: none.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.restart_dagster();

    suspend_services

    • Signature: 

      suspend_services()
      Suspends the core.dagster and core.postgres services. Compute pools are left running.

    • Arguments: none.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.suspend_services();

    suspend_compute_pools

    • Signature: 

      suspend_compute_pools()
      Suspends the app control, code-location, and run compute pools. Services are not explicitly suspended first.

    • Arguments: none.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.suspend_compute_pools();

    suspend_app

    • Signature: 

      suspend_app()
      Suspends core.dagster, core.postgres, and all Dagflow compute pools.

    • Arguments: none.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.suspend_app();

    Service Logs

    service_containers

    • Signature: 

      service_containers(service_name STRING)
      Shows containers for a service in the app core schema.

    • Arguments:

    Argument Required Description
    service_name Yes Core service name without schema, for example dagster or postgres. Must start with a letter and contain only letters, numbers, and underscores.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.service_containers('dagster');

    service_logs

    • Signature: 

      service_logs(
  service_name STRING,
  container_name STRING,
  line_count NUMBER
)
      Returns logs from a service container in the app core schema.

    • Arguments:

    Argument Required Description
    service_name Yes Core service name without schema, for example dagster or postgres. Must start with a letter and contain only letters, numbers, and underscores.
    container_name Yes Container name. Must start with a letter and contain only letters, numbers, underscores, or hyphens.
    line_count No Number of log lines to return. Defaults to 100 when NULL.

    • Required role: APP_ADMIN.

    • Example: 

      CALL dagflow_app.app_public.service_logs('dagster', 'webserver', 500);
CALL dagflow_app.app_public.service_logs('postgres', 'postgres', NULL);
      Use these procedures when the UI is unavailable, a code update is missing, or Snowflake reports an unhealthy service.

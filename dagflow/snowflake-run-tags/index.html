    Snowflake Run Tags

    This page is for users who launch runs and admins who grant Snowflake access. Set these Dagster run tags only when a run needs custom worker resources, external access integrations, or Snowflake secrets.

    CPU and Memory

    Use snowflake/cpu and snowflake/memory to request CPU and memory for one run.

    By default, run workers use 0.25 vCPU and 512Mi memory. This matches Dagster ECS' default Linux run-task resources. If dagster.yaml sets launcher-level run_resources, these tags override them for the run.

    Example:

    {
  "snowflake/cpu": "0.5",
  "snowflake/memory": "2Gi"
}

    CPU values must be positive vCPU quantities such as 0.5, 1, or 500m. Memory values must include a Snowflake-supported unit: M, Mi, G, or Gi.

    External Access Integrations

    Use snowflake/external_access_integrations when a run needs outbound network access allowed by a Snowflake external access integration.

    The value can be a comma-separated string:

    OPENAI_EAI,SLACK_EAI

    Or a JSON string array:

    ["OPENAI_EAI", "SLACK_EAI"]

    Each value must be an unquoted Snowflake identifier.

    The installed application must have USAGE on every integration used by the run:

    GRANT USAGE ON INTEGRATION OPENAI_EAI TO APPLICATION dagflow_app;

    Secrets

    Use snowflake/secrets when a run needs Snowflake secrets as environment variables. The value must be a JSON string array:

    [
  {
    "snowflakeSecret": "DAGFLOW_CONSUMER_DB.EAI_PROBE.API_TOKEN",
    "secretKeyRef": "secret_string",
    "envVarName": "MY_API_KEY"
  }
]

    Fields:

    • snowflakeSecret: Snowflake secret name, unqualified or dotted, such as DATABASE.SCHEMA.SECRET_NAME.
    • secretKeyRef: key exposed by the Snowflake secret, such as secret_string or password.
    • envVarName: environment variable name inside the run worker.

    The installed application needs USAGE on the secret schema and READ on each secret:

    GRANT USAGE ON DATABASE DAGFLOW_CONSUMER_DB TO APPLICATION dagflow_app;
GRANT USAGE ON SCHEMA DAGFLOW_CONSUMER_DB.EAI_PROBE TO APPLICATION dagflow_app;
GRANT READ ON SECRET DAGFLOW_CONSUMER_DB.EAI_PROBE.API_TOKEN
  TO APPLICATION dagflow_app;

    Secret injection is independent from external access integrations. If a run sets both tags, Snowflake validates whether the integration allows the selected secrets.

    Example Dagster Launch Tags

    When launching a run through Dagster GraphQL or any Dagster API that accepts execution metadata, pass tags as strings:

    [
  {
    "key": "snowflake/cpu",
    "value": "0.5"
  },
  {
    "key": "snowflake/memory",
    "value": "2Gi"
  },
  {
    "key": "snowflake/external_access_integrations",
    "value": "OPENAI_EAI"
  },
  {
    "key": "snowflake/secrets",
    "value": "[{\"snowflakeSecret\":\"DAGFLOW_CONSUMER_DB.EAI_PROBE.API_TOKEN\",\"secretKeyRef\":\"secret_string\",\"envVarName\":\"MY_API_KEY\"}]"
  }
]

    Dagflow validates tag values before creating the run-worker job service. Invalid resources, identifiers, JSON, duplicate environment variable names, or missing required fields fail the launch.

    RBAC

    Dagflow uses Snowflake application roles to control Dagster operations. Admins grant these roles to Snowflake account roles; users do not manage them directly.

    Roles

    Application role Dagflow role Can do
    APP_VIEWER VIEWER Open the UI and read Dagster state.
    APP_LAUNCHER LAUNCHER Launch, reexecute, terminate, and delete runs.
    APP_EDITOR EDITOR Manage schedules, sensors, backfills, assets, dynamic partitions, concurrency, and existing code-location images.
    APP_ADMIN ADMIN Add, remove, reload, or shut down code locations and call app admin procedures.

    Admin grant example:

    GRANT APPLICATION ROLE dagflow_app.APP_VIEWER TO ROLE analyst;
GRANT APPLICATION ROLE dagflow_app.APP_LAUNCHER TO ROLE service_runner;
GRANT APPLICATION ROLE dagflow_app.APP_EDITOR TO ROLE data_engineer;
GRANT APPLICATION ROLE dagflow_app.APP_ADMIN TO ROLE platform_admin;

    Choosing Roles

    • Give APP_VIEWER to users who inspect jobs, assets, run history, and logs.
    • Give APP_LAUNCHER to users who launch or stop runs.
    • Give APP_EDITOR to users who manage schedules, sensors, backfills, concurrency, or existing code-location images.
    • Give APP_ADMIN only to Dagflow operators.

    Operation Mapping
